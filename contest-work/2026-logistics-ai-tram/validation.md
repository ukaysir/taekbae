- 판정: 현재 근거로는 비권고

### 2. 차별화하여 수정

- 수정 방향:
  - 어려운 명칭인 `근거제약형 AI 모듈`보다 사용자가 받는 결과인 `배송경로 사전점검 정보`를 먼저 설명
  - 주 사용자를 택배 영업소·도시배송 배차담당자로 고정
  - AI는 공지 읽기·항목 정리·근거 연결·보류 요청까지만 담당
  - 상가 수는 실제 물량이 아니라 실제 경로·물량을 먼저 확인할 지역을 정하는 보조지표로 제한
  - 효과 주장은 확인시간·수정량·경로 일치율의 향후 시험으로 분리
- 장점: 기존 분석을 보존하면서 공식 평가의 주제·데이터·분석과정을 직접 회수할 수 있다.
- 약점: 실제 사용자의 문제 경험과 AI 미니시험이 아직 없다.
- 판정: 권고

- 단점: `근거제약형`, `근거 게이트`, `잠재 배송노출`, `TMS 앞단` 같은 용어가 한꺼번에 나오고 같은 논리가 반복돼 핵심을 이해하기 어렵다. 서비스 AI가 아직 시험되지 않았는데 기술 명칭이 성과처럼 보일 수 있다.

## 사용자 인터뷰

### Q1. 문제 경험

- 질문: 트램 공사 때문에 배송경로를 확인하거나 변경하는 문제를 직접 겪었거나 주변에서 본 적이 있는가?
- 사용자 답변: `아직 모름`
- 경험 분류: `아직 모름`
- 반영:
  - 배차담당자의 반복 확인과 불편을 사용자의 직접 경험처럼 쓰지 않는다.
  - 현재 사용자 문제와 수요는 `[미확인]`으로 유지한다.
  - 공식 공사자료의 형식 차이와 실제 데이터 결합 과정에서 확인한 작업량은 문제 존재의 제한적 근거로 사용하되, 현장 빈도·불편 정도·도입 의향은 별도 검증 대상으로 둔다.

### Q2. 반드시 보존할 핵심

- 질문: 현재 아이디어에서 반드시 보존하고 싶은 핵심은 무엇인가?
- 사용자 답변: `공모전에 맞게 아이디어는 유동적으로 선택`
- 분류: `사용자 제공`
- 반영:
  - 트램, 배송경로, 근거제약형 AI 등 현재 구성요소를 필수 보존 대상으로 두지 않는다.
  - 공식 평가기준 적합성, 실제 검증자료, 차별성, 실행 가능성이 더 높은 대안이 있으면 전환할 수 있다.
  - 기존 분석에 투입된 작업량만을 이유로 아이디어를 유지하지 않는다.

### Q3. 사용자와 AI의 실제 기여

- 질문: 현재 초안에서 사용자가 직접 결정하거나 수정한 부분은 무엇인가?
- 사용자 답변: `아이디어 선택은 사용자가 했고, 나머지는 전부 AI`
- 분류: `사용자 제공`
- 반영:
  - 데이터 수집·분석·결과 해석·문장 작성에 사용자가 직접 수행했다고 쓰지 않는다.
  - 기존 분석 과정 보고서의 `참가자와 AI의 공동 검증`, `참가자가 결과 해석과 최종 제출 판단` 같은 표현은 현재 답변과 충돌하므로 그대로 사용할 수 없다.
  - 공식 FAQ가 AI 결과의 그대로 제출을 경고하므로, 사용자가 핵심 주장과 수치를 직접 확인하고 수정 의견을 남기는 절차가 필요하다.
  - 최종 팀 기여 내용은 실제 참여가 추가된 뒤 그 범위만 기록한다.

### Q4. 이해하기 어려운 부분

- 질문: 현재 초안에서 가장 이해하거나 설명하기 어려운 개념은 무엇인가?
- 사용자 답변: 첫 문장부터 이해되지 않으며 `근거제약형 AI`라는 표현도 처음 보는 사람 입장에서 이해하기 어렵다.
- 분류: `사용자 제공`
- 반영:
  - 제목과 첫 문장에서 기술명을 앞세우지 않는다.
  - 첫 문장은 `사용자 → 문제 → 제안이 하는 일`의 순서로 작성한다.
  - `근거제약형 AI`는 필수 명칭으로 사용하지 않는다. 필요할 때는 `AI가 원문에 있는 내용만 정리하고, 애매하면 사람에게 확인을 요청하는 방식`처럼 작동 과정으로 먼저 설명한다.
  - `근거 게이트`, `잠재 배송노출`, `TMS 앞단` 같은 용어도 최초 등장 시 일상어로 풀거나 삭제한다.

- 분류: `아직 모름`
- 반영:

  - 5건·18행·451행·78,607행·69·12·994 같은 수치는 AI 작업 산출물로 분류하고, 사용자의 직접 확인 전에는 참가자 검증 완료로 표현하지 않는다.
  - 제출 전에 핵심 공식자료, 결과표, 해석을 사용자가 확인하고 확인 범위와 수정 내용을 기록해야 한다.

### Q6. 제출 전 사용자 참여 가능 범위

- 질문: 제출 전까지 직접 확인하고 판단하는 작업에 어느 정도 시간을 사용할 수 있는가?
- 사용자 답변: 초안 작성본을 확인하고 이해하는 데 제출 전까지 시간을 사용할 수 있음
- 분류: `사용자 제공`
- 반영:
  - 쉬운 한국어 초안과 함께 핵심 주장·수치·출처를 연결한 사용자 확인표를 제공한다.
  - 사용자는 초안의 의미, 주요 수치, 과장 여부, 본인이 동의하는 판단을 확인하고 수정 의견을 남긴다.
  - 실제 확인이 끝난 범위만 AI 활용기록과 팀 기여 내용에 반영한다.

### Q7. 가장 믿기 어려운 가정

- 질문: 현재 아이디어에서 가장 믿기 어렵거나 실제로 될지 의심되는 부분은 무엇인가?
- 사용자 답변: `실제 현장 사용`
- 분류: `사용자 제공`
- 반영:
  - 배차담당자가 현재 어떤 방식으로 공사정보를 확인하는지, 제안 정보를 사용할 의향이 있는지, 확인시간이 줄어드는지는 `[미확인]`으로 유지한다.
  - 실제 TMS 도입·연동·경로 개선을 핵심 성과로 쓰지 않는다.
  - 현장 사용을 전제로 하는 서비스안보다 현재 데이터로 직접 증명할 수 있는 분석안에 우선순위를 둔다.

## 사용자 인터뷰 요약

- 문제 경험: 아직 모름
- 반드시 보존할 요소: 없음. 공모전 적합성에 따라 변경 가능
- 현재 사용자 기여: 아이디어 선택
- 데이터·분석·해석·문장: AI가 수행
- 이해도 문제: 첫 문장과 `근거제약형 AI`부터 어려움
- 직접 확인한 근거: 없음
- 제출 전 참여: 쉬운 초안을 읽고 이해·확인·수정 가능
- 가장 취약한 가정: 실제 현장 사용

따라서 참가자의 직접 경험이나 현장 수요를 만들어 쓰지 않는다. 최종 방향은 사용자가 직접 이해하고 확인할 수 있고, 현재 보존된 데이터와 결과로 방어할 수 있어야 한다.

## 최종 진단

- 판정: `차별화하여 수정`
- 기존 아이디어 폐기 여부: 폐기하지 않음
- 이유:
  1. 공식 FAQ가 트램 공사 구간 배송 지연 분석을 허용한다. [S04]

  3. 계획공사·차로통제 정보는 기존 서비스에도 있어 현재 제목과 기술명만으로는 독창적이지 않다. [S10][S11][S13][S14]
  4. 가장 취약한 현장 사용 가정을 축소하고 데이터 분석 중심으로 바꾸면 기존 근거를 보존하면서 공식 배점 중 데이터 활용 20점·분석과정 30점에 더 직접 대응할 수 있다.

## 1차 승인용 아이디어 3개

### A. 배송경로 사전점검표

- 해결하려는 문제: 배차담당자가 지역 공사공지의 장소·방향·차로통제를 예정 배송경로와 바로 대조하기 어렵다.
- 대상: 택배 영업소·도시배송 배차담당자
- 핵심 해결 방식: AI가 공지를 쉬운 항목으로 정리하고, 사람이 확인한 도로구간·현재 교통·주변 상가 분포를 한 장의 점검표로 제공한다.
- 실제 작동 과정: 공지 수집 → AI 항목 정리 → 도로 후보 생성 → 사람이 근거 확인 → 점검표 생성 → 담당자가 기존 경로와 대조
- 기존 방식과의 차이: 새 경로최적화 시스템을 만드는 것이 아니라 대전의 부분차로 공사를 근거와 함께 기존 경로 검토에 넣는다.
- 사용자 경험·자원과의 연결: 직접 현장 경험은 없지만 기존 5건·18행·451행·상가 분석을 활용할 수 있다.
- 필요한 근거와 데이터: 실제 비식별 배송경로, 배차담당자 인터뷰·사용성 시험
- 가장 취약한 가정: 담당자가 실제로 점검표를 사용할 것이라는 가정
- 예상 심사 반론: “카카오·Waze 같은 서비스로 충분하지 않은가?”, “실제 TMS와 대조하지 않았는데 현장성을 어떻게 증명하는가?”
- 공모전 적합성: 높음
- 독창성: 중간
- 현실성: 중간
- 수상 경쟁력: 중간

### B. 트램 공사 물류영향 공지표준

- 해결하려는 문제: 공사공지에는 사람이 읽는 장소명이 있고 지도·배차시스템은 방향별 도로구간을 사용해 서로 바로 연결되지 않는다.
- 대상: 대전시·트램 시행기관의 공사·교통정보 담당자와 외부 지도·물류정보 사업자
- 핵심 해결 방식: 공사장소, 방향, 기간, 통제차로, 표준 도로구간, 원문 근거, 연결 신뢰도를 담은 공통 공지항목을 제안한다.
- 실제 작동 과정: 공지 작성 → AI 구조화 보조 → 표준 링크 후보 → 담당자 승인 → 공개 데이터 파일 배포 → 지도·물류서비스가 선택적으로 활용
- 기존 방식과의 차이: 배송경로를 직접 계산하지 않고, 여러 서비스가 재사용할 수 있는 공사정보의 입력 형식을 개선한다.
- 사용자 경험·자원과의 연결: 18개 후보 중 10개 결합·6개 재검증·2개 차단 결과가 형식 불일치의 실제 사례다.
- 필요한 근거와 데이터: 대전시 현행 데이터 제공절차, 담당부서 인터뷰, 외부 서비스의 수용 필드
- 가장 취약한 가정: 공공기관이 새 형식을 운영하고 외부 사업자가 사용할 것이라는 가정
- 예상 심사 반론: “물류 아이디어보다 행정 데이터 표준 제안에 가깝지 않은가?”, “Waze 등 기존 입력방식과 무엇이 다른가?”
- 공모전 적합성: 중간
- 독창성: 중간
- 현실성: 중간

### C. 트램 공사구간 배송 점검 우선순위 분석

- 해결하려는 문제: 여러 공사구간 중 실제 배송경로와 물량을 먼저 확인해야 할 곳을 정할 근거가 부족하다.
- 대상: 대전 지역 물류·배차 담당자와 교통대책 담당자
- 핵심 해결 방식: 공사범위의 확실성, 현재 교통상태, 주변 영업 중 상가 분포를 각각 보여주고 우선 확인할 구간을 정한다.
- 실제 작동 과정: 공사정보 5건 정리 → 도로 후보 18행 검증 → 신뢰도에 따라 10개만 분석 → 교통관측 451행 결합 → 공사 3건의 250m 상가 분포 비교 → 우선 확인구간 제시
- 기존 방식과의 차이: 새로운 AI 서비스나 경로최적화를 주장하지 않고, 공사정보가 불완전할 때도 근거 수준을 공개하며 분석 가능한 구간만 비교한다.
- 사용자 경험·자원과의 연결: 현재 보존된 분석결과를 가장 직접적으로 사용하며 사용자가 제출 전 표와 출처를 확인할 수 있다.
- 필요한 근거와 데이터: 실제 배송량이 없다는 한계를 명시하고, 가능하면 지역 우편·화물 통계처럼 물류 관련 데이터를 보조로 추가 검토
- 가장 취약한 가정: 상가 분포가 실제 배송활동을 충분히 대리한다는 가정
- 예상 심사 반론: “상가 수가 배송량이 아닌데 왜 물류 영향이라고 부르는가?”, “짧은 교통관측으로 미래 대응을 말할 수 있는가?”
- 공모전 적합성: 높음
- 독창성: 중간
- 현실성: 높음
- 수상 경쟁력: 높음

다만 상가 수는 실제 배송량이 아니므로 `배송 영향 점수`를 확정값처럼 만들지 않는다. 공사범위·현재 교통·상가 분포를 분리해 보여주고 `우선 확인 대상`으로만 해석한다.

## 선택안 C 검증

### 한 문장 정의

대전 트램 공사 지역별로 공사범위, 현재 차량 속도, 주변 상가 분포를 비교해 실제 택배 운행을 먼저 조사할 지역을 찾는 분석이다.

### 처음 읽는 사람을 위한 설명

대전 트램 공사로 일부 도로의 차로가 줄어들고 있다. 그러나 어느 공사가 실제 택배 운행을 늦추는지는 아직 확인되지 않았다. 이 분석은 공사 지역별 교통상태와 주변 상가 분포를 비교해 실제 택배 경로와 배송시간을 먼저 조사할 지역을 찾는다. 도로를 새로 선택하거나 우회경로를 만드는 분석은 아니다.

### 작동 범위

1. 대전시가 공개한 트램 공사 위치·기간·차로통제를 정리한다.
2. 공사 위치와 국가표준 도로구간의 연결 후보를 만든다.
3. 근거가 충분한 후보만 현재 차량 속도 비교에 사용한다.
4. 공사 주변의 영업 중 상가 분포를 별도 지표로 계산한다.
5. 공사범위·교통상태·상가 분포를 한 점수로 합치지 않고 나란히 제시한다.
6. 후속 조사에서는 우선지역의 실제 택배 경로와 배송시간을 수집해 공사 영향을 확인한다.

- 결론: `실제 택배 운행을 먼저 조사할 지역`까지는 제안할 수 있으나 `배송 지연 예측`, `최적경로 생성`, `현장 도입 효과`는 주장하지 않는다.

### 가장 큰 위험과 완화

| 위험 | 완화 방식 |
|---|---|
| 상가 수를 배송량으로 오해 | `주변 상가 분포`로만 표기하고 실제 배송량과 다르다고 같은 면에 명시 |
| 하루 관측을 평상시 교통으로 일반화 | 관측일·시간·수집 횟수를 공개하고 `현재 상태 예시`로 제한 |
| 도로 연결 오류 | 포함·재확인·제외 상태와 원문 근거를 공개 |
| 현장 사용을 이미 입증한 것처럼 보임 | 실제 배송자료 대조와 배차담당자 시험을 후속 검증으로 분리 |
| AI가 전부 수행한 결과를 참가자 작업으로 오해 | AI 활용내역을 공개하고 사용자가 실제 확인·수정한 내용만 참가자 기여로 기록 |

### 평가할 수 있는 지표

- 공사공지 1건당 도로 후보 수와 근거 확인 비율
- 분석 포함·재확인·제외 후보 수
- 교통관측의 수집 시각·성공률·결측률
- 공사 지역별 현재 속도와 기준 속도의 차이
- 공사 지역 250m 안 영업 중 상가 수
- 후속 검증 시 실제 배송경로와 공사구간의 중첩률
- 후속 검증 시 공사 전후 배송시간 변화

## C안 판정

- 최종 상태: `차별화하여 수정`
- 유지할 것: 공식 공사정보와 도로구간 연결, 근거 수준 공개, 현재 교통관측, 상가 공간분석
- 삭제할 것: `근거제약형 AI`를 앞세운 제목, 자동 경로판단, 30분 예측 성능, 현장 도입 효과, 상가 수를 배송량처럼 해석하는 표현
- 추가로 필요한 것: 사용자의 초안·핵심 수치·출처 확인, 가능하면 실제 배송자료 또는 현장 담당자 검증
