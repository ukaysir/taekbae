# 제출용 시각자료

제안서와 분석 과정 보고서에 삽입할 16:9 시각자료의 원본 HTML/CSS와 PNG를 관리한다.

## 구성

- `src/01_service_flow.html`: 공사공지에서 TMS 경로판단까지 이어지는 서비스 흐름
- `src/02_verification_workspace.html`: 읍내삼거리 공사정보·도로구간 근거 게이트
- `src/03_evidence_summary.html`: 분석결과가 배송판단으로 이어지는 근거 사슬
- `png/`: 위 HTML을 1600×900으로 캡처한 제출용 이미지

## 표현 원칙

- 실제로 확인한 수치만 사용한다.
- 화면은 개념 시안으로 표시하며 운영 중인 서비스로 표현하지 않는다.
- 18개 연결 후보는 시범 데이터 결합 10개, 담당자 재검증 6개, 연결 차단 2개로 표현한다.
- 451개 교통기록은 확정한 공사구간의 현재 교통상태를 반복 갱신할 수 있다는 근거로 사용한다.
- 반경 250m 상가 수는 실제 배차경로·물량을 먼저 대조할 공사영향권을 찾는 잠재 배송노출로 사용한다.
- 최종 경로 유지·우회는 기존 TMS의 예정경로 대조와 담당자 판단으로 표현한다.
