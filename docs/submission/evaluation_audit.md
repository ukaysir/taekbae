- 공식 제출 주제: **대전 트램 공사구간의 도심배송 위험 관측·예보 모듈**

## 제출 차단·확인 사항

### P0 / COMPLIANCE_RISK — 저작재산권 조항 충돌

- 공고문: 수상작의 홍보·전시 목적 2차 활용 권한을 주최기관에 부여하되 저작재산권은 원칙적으로 참가자에게 귀속
- 원본 HWP 서식 4 참가서약서 제4항: 출품작의 저작재산권을 주최기관에 이양
- 판정: 같은 공식 모집의 권리 귀속이 상충한다. 참가자가 원본 서약 문구를 임의 수정해서는 안 된다.
- 조치: 주최기관에 어느 조항이 적용되는지, 정정 서식이 있는지 서면 확인 후 서명한다.

### P0 / COMPLIANCE_RISK — 제출자 고유정보와 공식 HWP 부재

- 팀명·인적사항·동의·서명·추첨번호가 없고, 자동 HWP 편집은 원본 구조 보존 검증에 실패해 결과 파일을 폐기했다.
- 조치: 참가자가 정보를 확정한 뒤 원본 HWP에서 수동 입력하고 6쪽 전체를 다시 렌더링·비교한다.

### P1 / EVIDENCE_RISK — 모델평가 대기

- 현재 실측이 48시간·3개 날짜·288스냅숏 기준에 미달한다.
- 조치: 통합 최종화가 성공할 때만 성능표와 모델 파일을 사용한다. AI가 기준모델을 이기지 못하면 예측기능은 비활성으로 제출한다.

### P1 / COMPLIANCE_RISK — 추첨번호 표기 방식

- 공고문은 7개 번호에 ○ 표시, 원본 서식 5는 7개 번호를 빨간색 폰트로 표기하라고 한다.
- 조치: 정확히 7개를 빨간색으로 표시하고 ○ 처리하는 방향으로 준비하되, 주최기관 답변을 우선한다.

## 제출 직전 판정 기준

다음이 모두 증명된 경우에만 `조건부 제출 가능` 이상으로 바꾼다.

1. 공식 HWP와 PDF가 모두 열리고 제안서가 10쪽 이내이며 쪽번호가 있다.
2. 분석 과정 보고서 HWP와 PDF가 같은 내용이고 권장 3~5쪽을 충족한다.
3. 참가신청서·서식 4·서식 5의 필수 선택과 전원 서명이 완료됐다.
4. 서식 5에 정확히 7개 번호만 표시됐다.
5. 공공데이터 1건 이상, 모든 데이터·이미지 출처, AI 프롬프트 3개 이상과 수정 사례가 있다.
6. 저작재산권 충돌에 대한 주최기관 답변 또는 정정 서식을 보관했다.
7. ZIP 내부 파일과 이름을 확인했고 압축 후 50MB 미만이다.
8. PMS에서 최종 `신청` 후 접수증을 저장했다.
