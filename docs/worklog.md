## 1. 작업 경과

### 2026-07-14

1. 제공된 `붙임1.txt`, `붙임2.txt`를 기본 자료로 공모 분야, 제출조건, 평가항목, 선행 수상작을 파악했다. 원본 PDF/HWP는 추출본으로 판단하기 어려운 표·이미지·서식을 재확인할 때 사용할 자료로 보존했다.
2. 지정공모 5개와 자유공모 중 아이디어 적합성을 비교한 뒤 **지정공모 3 — 도시 변화에 따른 미래 물류 흐름 예측 및 대응 아이디어**로 방향을 확정했다.
3. 대전 트램을 아이디어의 첫 실증사례로 고정했다.
4. AI를 두 층으로 분리했다. 서비스에는 30분 후 통행시간 예측 모델을 사용하고, 생성형 AI는 조사·코드·검토·문서화 보조로 사용하되 결과를 사람이 검증하도록 했다.
5. API와 데이터가 실제 사용 가능한지 확인하기 위한 Gate 0~7을 설계했다. 현재는 명세와 후보 데이터 조사 단계이며 인증 호출·모델 학습은 아직 수행하지 않았다.
6. `출차시간`의 사용자를 재검토했다. 서비스가 출차시각을 직접 추천한다는 표현을 폐기하고, 배차담당자에게 예상 지연·위험 신호를 제공하는 구조로 수정했다.
7. 트램 공사 종료 후 지속성이 없다는 문제를 반영했다. 공사 전–중–개통 전환–사후평가–다른 계획공사로 이어지는 전주기 구조와 정직한 종료조건을 추가했다.
8. 택배 영업소나 도시배송업체의 TMS를 대체할 수 있는지 검토했다. 주문·송장·기사·차량·정산·경로최적화는 기존 시스템이 담당하고, 본 제안은 계획공사 위험정보를 제공하는 보완 모듈로 확정했다.
9. 위 결정을 반영해 [plan.md](../plan.md)를 v2로 전면 정리했다. 18개 절에 사용자, 역할 경계, 지속 구조, 데이터/API, 검증 게이트, 분석법, 연동 규격, 평가 대응, 일정, 위험, 완료 정의를 기록했다.
10. Git 저장소를 초기화하고 Git Credential Manager 웹 인증으로 `https://github.com/ukaysir/taekbae`에 최초 커밋을 푸시했다.
11. 이번 요청으로 [research.md](research.md)와 본 작업 기록을 생성해 지금까지의 조사·결정·미검증 항목을 동결했다.
