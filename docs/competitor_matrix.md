- 확인일: 2026-07-15
- 비교 원칙: 공개된 공식 기능만 `확인`으로 표시한다. 비공개 기능의 부재를 추정하지 않는다.

## 핵심 차별성

일반 경로최적화나 미래 교통예측만으로는 차별성이 없다. 카카오 TMS는 이미 실시간 도로상황과 미래 교통량을 반영한 배차와 ETA를 공개적으로 설명하고, 미래 길찾기 API는 도로공사를 포함한 전체차선 통제 옵션을 제공한다. 따라서 제안의 최소 차별 단위는 다음 네 가지가 함께 있을 때다.

1. 공사구간·기간·차로통제·방향을 별도 사건 테이블로 관리한다.
2. 사건을 표준 링크와 실제 관측구간에 근거·신뢰도와 함께 매핑한다.
3. 배송경로가 공사에 얼마나 노출되는지 현재 관측/향후 예보 신호로 반환한다.
4. 같은 구조로 공사 전 기준선, 공사 중 운영, 개통 후 사후평가를 잇는다.

이는 “더 좋은 TMS”가 아니라 기존 TMS가 알기 어려운 지역 계획공사 맥락을 붙이는 역할이다. 공개문서만으로 모든 상용 TMS의 내부기능을 알 수 없으므로 실제 도입 전에는 업체 인터뷰와 API 계약 검토가 필요하다.

## 실제로 줄이려는 수작업

배차담당자가 공사공지, 교통지도, 내부 엑셀을 각각 열어 경험적으로 버퍼를 정하는 과정을 다음 흐름으로 줄인다.

`기존 배차경로 CSV → 구간별 공사 노출·관측위험 결합 → 위험 구간 검토 → 기존 TMS에서 최종 배차 결정`

출차시각, 최종 경로, 주문 배분은 내부 운영조건을 모르는 본 서비스가 결정하지 않는다.

## 실증화 사업 7개 서비스와의 경계

공고문 별첨의 디지털 물류플랫폼·자동화기기·공모 신서비스는 배송접수, 물류거점, 자동화장비 등 구현사업 범위를 포함한다. 본 제안은 새 택배거점·자동화기기·주문플랫폼을 만들지 않고, 공개 교통·공사정보를 결합한 의사결정 데이터 모듈로 제한한다. 제출 전 원본 별첨 14~16쪽의 각 서비스명을 다시 대조한다.
