## 주제 선택 결론

트램 아이디어는 지정공모 3의 설명과 예시에 직접 맞고 가점 대상이다. 다만 “트램”만으로는 차별화되지 않는다. 2025년 수상작에 트램 심야 공동물류가 있고 상용 TMS에는 미래교통·배차기능이 있기 때문이다.

최종 차별 단위는 다음이다.

> 공사 이벤트의 구간·기간·차로통제·방향을 교통링크와 근거 기반으로 연결하고, 기존 배차경로에 공사 노출과 관측/예보 위험을 붙여 공사 전–중–후에 같은 지표로 평가하는 외부 모듈

## 공사 후 활용

공사 중에는 현재위험·향후 예보, 개통 전환기에는 기준선 재설정, 개통 후에는 지속 병목·예측오차 사후평가에 쓴다. 다음 도로공사·대규모 행사·도로공간 재편에 같은 이벤트–링크–관측 구조를 재사용할 수 있다.

새 사업이나 운영주체를 확보하지 못하면 트램 전용 서비스는 사후평가 후 종료한다. 영구서비스나 기관 도입을 약속하지 않는다.

## 공사–관측구간 매핑

공사 이벤트 5개와 후보 관측구간 18개를 기록했다.

- 동부여성가족원~영진로얄아파트: 공식 공사이미지와 2025-08-22 공지문으로 계족로 공사범위는 high 신뢰이나, 같은 도로·지점·방향 라벨이 두 번 나타나 개별 발생순번은 medium으로 보수적 포함
- 읍내삼거리: 공식 공사이미지와 화면 라벨이 일치하는 계족로 양방향 2개를 high 신뢰로 시범 포함
- 서대전역네거리~서대전네거리: 표준 링크 경로는 계백로로 확인했으나 화면의 동일 라벨 후보가 복수여서 medium
- 테미고개: 표준 링크상 인접도로는 충무로·테미로이며 기존 계룡로 추정은 폐기; 정확 지점 라벨이 없어 제외
- 나머지: 지명·도로 일부만 맞아 low 후보

공식 공사이미지 2건은 URL·HTTP 200·PNG 1125×1125·SHA-256 일치를, 공지문 1건은 HTTP 200과 기대문구 일치를 자동 검증했다. 이 근거로 공사범위가 high 신뢰인 2개 이벤트에 medium 이상 관측구간이 있어 Gate 2를 통과했다. 이는 중복 라벨의 개별 공간 위치까지 확정했다는 뜻이 아니다. 표준 링크 재현결과는 서대전 양방향 각 4링크, 654.016m/650.315m, 도로명 모두 계백로다.

## AI 분석 설계와 현재 결과

두 AI를 구분한다.

- 서비스 AI: 30분 뒤 링크 속도/통행시간을 예측할 비선형 회귀 후보
- 분석보조 AI: 아이디어·코드·조사·반론·문서 초안 보조

서비스 AI는 지속성, 같은 요일·시각 기준모델, 히스토그램 그래디언트부스팅 후보를 시간순으로 비교한다. 랜덤분할은 쓰지 않는다. 합성자료 테스트는 코드가 작동한다는 증거일 뿐 실제 성능 근거가 아니다.

현재 최소조건은 288스냅숏, 48시간, 3개 날짜, 5,000예제다. 실제 자료가 미달하여 `model_evaluation.json`은 `not_evaluated`, `insufficient_data`다. 허용 주장은 수집기 동작과 현재 관측 모니터링뿐이다.

## 남은 검증

1. 공공데이터포털에서 최신 대전교통·KMA 활용신청과 인증키 상태를 사용자가 확인한다.
2. 수집 48시간·3일 기준을 채운 뒤 자동 최종화 명령으로 실데이터 모델과 동결 해시를 확정한다.
3. 사용자가 팀정보·연락처·동의·서명·심사위원 7개 번호를 제공한다.
4. 제안서·보고서 HWP/PDF를 최종 페이지 제한에 맞춰 검수한다.
5. 제출 직전 공고 정정, 트램 일정·공사현황, URL, ZIP 50MB를 재검증한다.

이 항목이 끝나기 전까지 도입성과, 절감률, 실제 배송지연 인과효과, 예측 우월성은 UNKNOWN이다.
